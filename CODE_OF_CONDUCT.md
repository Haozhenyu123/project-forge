# Code of Conduct

Project Forge follows a simple standard: be respectful, constructive, and evidence-oriented.

Contributors should:

- discuss ideas and tradeoffs without personal attacks;
- document assumptions and sources when making architecture claims;
- avoid using Project Forge to fabricate evidence or overstate confidence;
- keep implementation workflow responsibilities delegated to Superpowers or the receiving implementation system.

Maintainers may remove comments, issues, or contributions that violate this standard.
